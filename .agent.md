### Objective
Build a professional, minimal Android app combining Discord + Quora + Clubhouse with creator economy.

- **Design**: Dark theme, Glassmorphism, Minimal UI

### Multi-Platform Backend-First Architecture Rules
- 🚨 **Highest Priority Rule:** Always design the backend (Supabase DB, RLS, Edge Functions, APIs) to support both the Mobile App and the Official Website (including User, Creator, Author, Employee, Moderator, Admin, Super Admin, Verification, and Support portals) from day one. Never build mobile-only or frontend-coupled architecture.
- 🚨 **Backend Business Logic:** Keep business logic inside the backend. Use a clean REST or RPC API architecture. Never write frontend-specific business logic or use mock/fake services.
- 🚨 **Pre-Implementation Check:** Before implementing any feature, ask:
  1. Will this work on both the mobile app and website?
  2. Will this work for Admin, Employees and Users?
  3. Will this scale to 1M+ users?
  4. Will this require database changes later?
  5. Is this secure?
  6. Is this reusable?
  7. Is there a better long-term architecture?
  If any answer is NO, redesign it before writing code.
