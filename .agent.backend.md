# Creania Backend Integration Agent Configuration (v2.0)

This document is a dedicated agent configuration and guideline file specifically for implementing the cross-platform, backend-first database and API system using **Supabase** and **LiveKit**.

---

## 🏗️ Core Architecture & Design Constraints

1. **Multi-Platform Ready:** Do not write mobile-only or local-only business logic. The PostgreSQL schemas, RLS rules, and Edge Functions must support both the Mobile App and the Web Platforms (User, Creator, Author, Admin, Employee, Moderator, Support, and Verification portals).
2. **Backend-First Implementation:**
   - 1. Postgres Migrations, Tables & Triggers.
   - 2. Row Level Security (RLS) & RPC Functions.
   - 3. Flutter client state integration.
3. **No Mocks or Hardcoded Values:** Implement actual transactional databases and secure endpoints.
4. **Pre-Implementation Verification:** Always ask the 7-question scale, security, and reusability checks before writing code.

### 💬 Phase 2.4: Realtime E2EE Private Messaging System

Build a secure, client-side encrypted chat interface synchronized across devices.

#### 📝 Step-by-Step Instructions:
- [ ] **Step 11: Client-Side E2EE Cryptography Engine**
  - Implement the cryptographic flow inside a new service:
    1. **Key Generation:** Generate X25519 public/private keys for the user. Store private key securely in local device memory, and publish public key to a column in `profiles`.
    2. **Key Exchange:** Pull the recipient's public key from `profiles`. Compute the shared secret using Diffie-Hellman.
    3. **Key Derivation:** Pass the secret to HKDF-SHA256 to generate a 256-bit symmetric key.
    4. **Symmetric Encryption:** Encrypt the message payload using AES-256-GCM.
- [ ] **Step 12: Realtime Message Stream Sync**
  - Replace current mock chat streams with Supabase Realtime Channels.
  - Subscribe to changes in the `messages` table for the target `room_id` or `receiver_id`.
  - Decrypt the encrypted payload locally on the client using the shared symmetric key and render the decrypted string in the UI.
